Contexto del Proyecto: Sistema de Gestión de Reservas

Curso: Desarrollo de Interfaces 2 - Evaluación parcial.
Objetivo General: Desarrollar una aplicación web funcional que permita a los restaurantes gestionar reservas de mesas, solucionando problemas actuales de duplicaciones y mala organización en horarios de alta demanda.
Modalidad: Trabajo grupal de 3 integrantes.
+2

Funcionalidades Principales a Desarrollar:
Registrar reservas de mesas de forma centralizada para evitar conflictos en los horarios.

Gestionar los horarios de ocupación previniendo la duplicación de reservas.

Implementar un calendario interactivo que muestre las reservas realizadas para facilitar la administración.

Proveer una interfaz intuitiva y accesible (responsive) tanto para dispositivos móviles como de escritorio.
+2

Atender a dos tipos de usuarios: administradores del restaurante (gestión) y clientes (visualización de disponibilidad y creación de reservas).
+2

Requerimientos Tecnológicos:

Diseño (Figma): Creación de wireframes y prototipos funcionales avanzados que simulen la interacción real del sistema, incluyendo transiciones, animaciones, selección de fechas y confirmación de reservas.
+2


Desarrollo (React): Configurar un entorno de desarrollo moderno utilizando React.


Componentes: Crear componentes básicos y reutilizables usando JSX (ej. formulario de reservas, calendario interactivo, lista de reservas).
+3


Gestión de Estado (Hooks): Aplicar useState para manejar el estado de las reservas y useEffect para actualizar el calendario y evitar conflictos en tiempo real.
+2

Entregables Requeridos:

Diseño: Enlace compartido de Figma con wireframes y prototipos interactivos.
